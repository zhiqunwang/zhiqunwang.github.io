一、bean注册相关

@Configuration

Java Bean的配置来源标识，与XML配置异曲同工。

@Bean

具体bean配置，默认方法名称为Bean的name，可通过value指定多个BeanName

init-method、destroy-method指定初始化、销毁方法。

@Scope

bean在容器中的存在形式，默认singleton。（容器在初始化时进行预加载）

可选类型：prototype（原型，每次请求获取的bean都是新创建的）、request、session

@ComponentScan

扫描注册类，定义bean扫描规则。

value:指定扫描的包路径

includeFilters:指定扫描的时候按照什么规则包含哪些组件

excludeFilters:指定扫描的时候按照什么规则排除哪些组件

其中includeFilters，excludeFilters可自定义过滤器，自定义过滤器需实现TypeFilter接口。

@Lazy

针对singleton而说，当加上该注解后，容器启动时不进行预加载，而是第一次应用索要bean时进行加载。

@Conditional

使用在类上或者方法上，定义bean的注册条件，条件规则器实现Condition接口。

@Import

快速给容器中导入组件,导入后在容器中组件的name 是导入组件的全类名。

@Import(value={Color.class,MyImportSelector.class,MyImportBeanDefinitionRegistrar.class})

三种类型的导入方法：

  • 导入具体组件的class类
  • 实现ImportSelector接口，导入该实现类，返回需要导入组件的全类名集合
  • 实现ImportBeanDefinitionRegistrar接口registerBeanDefinitions方法，在该方法内手动进行注册组件的BeanDefinition

@ImportResource

导入beanDefinition定义的xml文件，即描述bean的xml文件

总结

给容器中注册bean的方式：

  • @ComponentScan + @Controller、@Service、@Reporstory、@Componenet
  • @Bean 方式导入三方组件
  • @Import快速导入三方组件
  • @ImportResource 导入定义描述bean的xml文件

二、bean生命周期相关

指定初始化和销毁方法

@Bean指定init-method,destroy-method,单实例是在容器关闭时调用destroy方法，而多实例则不会管理这个bean销毁

实现spring bean生命周期相关接口

bean 实现InitializingBean接口afterPropertiesSet 方法执行初始化，实现DisposableBean接口destroy方法进行销毁前的执行

使用JSR-250注解

@PostConstruct、 @PreDestroy

实现BeanPostProcessor

实现BeanPostProcessor，对所有bean创建后，init-method之前、之后对bean进行包装等操作。

实现xxxAware接口

spring内部将进行值的回调设置。比如ApplicaitonContextAware,BeanNameAware,EmbeddedValueResolverAware

三、bean依赖注入相关

@Value

  • @Value(“张三”)
  • @Value(“#{20-2}”)
  • @Vaule(“${}”) 读取配置文件中的值（在运行变量中的值）

@PropertySource

@PropertySource(“classpath:/person.properties”) 将配置文件中的值导入环境变量

@Autowired

默认按照组件的类型去匹配，类型匹配到多个然后按照name去匹配。默认必须可以找到匹配的值，可通过@Autowired(required = false)设置非必须注入，可标注在方法、属性、参数上

@Qualifier

指定匹配的模式按照bean的name匹配

@Primary

spring装配的时候默认使用首选的bean，当一个类型匹配多个实例后，优先级的选择。

@Resource

JSR-250提供，默认按照组件名称进行装配,@Resource(name=””)可指定name.但是没有按照类型同时按照优先级进行匹配

@Inject

@Profile

@Profile(“dev”) 自动装配环境搭建，默认default
